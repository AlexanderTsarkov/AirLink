# AirLink Documentation Map

This directory contains durable project documentation.

## Current Structure

- `product/` — accepted product documentation, current state, vision, and product WIP.
- `chatgpt/` — minimal stable context for using ChatGPT in the project.

The active iteration context is stored in the repository root as `ITERATION.md`.

## Documentation Status

- Canonical documents describe accepted project truth.
- WIP documents are drafts and are not implementation requirements.
- Legacy Google Drive and local project materials are source material only.
- Temporary exports, comparisons, and scratch work belong in local `_working/`, which is not committed.

New documentation directories should be added only when a real retained document requires them.
