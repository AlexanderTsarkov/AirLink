# Product Work in Progress

This directory contains product drafts, competing formulations, unresolved hypotheses, and working specifications that are still under discussion.

## Rules

- WIP is not canonical product truth.
- WIP is not an implementation requirement unless a task explicitly authorizes a defined part of it.
- Each WIP document should state its purpose, status, sources, assumptions, open questions, and expected promotion target.
- Facts, decisions, assumptions, hypotheses, and unresolved questions must be distinguishable.
- Legacy content must be evaluated and rewritten, not pasted and silently accepted.
- Once accepted, durable content should be promoted into the appropriate canonical document.
- Obsolete WIP should be removed or deliberately retained with a clear historical reason.

Temporary exports and disposable intermediate work belong in local `_working/`, not here.
